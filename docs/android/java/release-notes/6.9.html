Scandit Data Capture SDK 6.9 Release Notes (August 5th 2021)#

See also the System Requirements of the Scandit Data Capture SDK across all platforms.

New Features#

  • Increased the minimum Android version to 5.0 aka API 21 (Lollipop).

  • Added a new data capture mode, BarcodeSelection. For more information, check our Get Started With Barcode Selection guide.

  • Added LabelCaptureSettings.locationSelection.

  • Allow expired certificates via AnalyticsSettings.setProperty with the key allowExpiredCertificates.

Behavioral Changes#

  • Improved regex for Swiss DL MRZ in ID Capture.

  • When mrzFullScreen recognizer is used and IdCaptureOverlay is using any MRZ layout the scan area margins are set to 0 dp.

Performance Improvements#

  • Improved scanning performance for (Micro-)QR, (Micro-)PDF417, and Data Matrix codes.

Bug Fixes#

  • Fixed rare MSI Plessey false positives.

  • Fixed a problematic camera state that could be reached if the camera experienced an error while running. The camera now properly turns off and tries to restart again until it succeeds.

  • Fixed the crash when setting the frame source to null while listening to the DataCaptureContextListener.

  • Fixed a crash caused by a rare failure to configure the camera.

  • Fixed a crash when reusing augmentation views in BarcodeTrackingAdvancedOverlay.

  • Fix the rare crash when recreating IdCaptureOverlay.

  • Fixed an issue that IdCaptureOverlay was sometimes not drawn when IdCapture was recreated.

  • HIBC parser: fixed the issue that codes with space check digit were not parsed.

  • Fixed an issue that context status error with code 1027 was shown even if newly attached IdCapture contained required libraries.

  • Fix an issue that passports and alien passport were scanned for IdDocumentType.ID_CARD_VIZ settings.

  • Fixed an OutOfMemoryException when reopening the scanner about a hundred times.

  • Fixed an issue that event listener was not registered.

Updates#

Scandit Data Capture SDK 6.9.1 Bug Fixes (August 23rd 2021)#

  • Fixed a memory leak in the DataCaptureView that can cause the view hierarchy to leak (introduced with 6.9.0).

  • Fixed a rare crash that could occur when switching between user-facing and world-facing cameras.

  • Fixed an issue that BarcodeSelectionAutoSelectionStrategy was not working correctly when selection strategy was set after selection type applied to settings.

  • GS1 AI parser: Fixed an issue where the day was incorrectly parsed for fields with Application Identifier equal to 11, 12, 13, 14, 15, 16 or 17.

  • Fixed an issue that private (hidden) DataCaptureView.SetLogoHidden method couldnâ€™t disable drawing logo for IdCaptureOverlay.

  • License check errors caused by switching to unlicensed text recognizers are properly reported by the context.
